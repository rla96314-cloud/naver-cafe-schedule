# 재기획 v2 — 카페 대문 주간 스케줄 도구

2026-07-06. 전체 재검토(문제 정의부터) + 사용자 확장(나 혼자 → 운영진 몇 명 공유).
v1 설계문서(`~/.gstack/projects/.../design-*.md`)와 달리, 이 문서의 제약은 전부 **실측 결과**다.

---

## 1. 문제 재정의

**누가**: 스트리머 그룹 카페 운영진 여러 명 (기술 수준 제각각, PC도 제각각)
**무엇**: 매주 바뀌는 주간 스케줄표를 네이버 카페 대문에 올린다. 칸을 누르면 그 방송으로 이동.
**왜 도구가 필요한가**: 매주 포스터를 손으로 만드는 노가다 제거 + 링크 클릭 가능하게.

**v1과 달라진 핵심**: "내 PC의 도구"가 아니라 **"운영진 누구나 여는 도구"**.
→ 이 한 줄이 아키텍처를 바꾼다. (아래 §4)

## 2. 확정된 사실 (전부 실측)

### 네이버 대문 생존 규칙 (스파이크 1·2차, 실제 카페에서 확인)
| 마크업 | 결과 |
|---|---|
| `<a>` 텍스트 링크 (color/밑줄만) | ✅ 생존 |
| 배경색·border-radius·box-shadow·padding | ✅ 생존 |
| 외부 `<img src=https>` | ✅ 생존 |
| `data:` base64 이미지 | ❌ 삭제 |
| **`<a>`에 background나 display 부여** | ❌ **요소째 삭제** → 카드 전체 클릭 불가능 |
| div `height` | ⚠️ **미검증** (P0에서 확인) |

### 시트 구조 (실제 시트에서 확인)
- 탭 1개 = 1주. 라벨 행("0615 ~ 0621") + 요일 헤더 행 + 멤버당 시간행/제목행.
- **제목을 안 적으면 제목행 자체가 없음** (파서가 이걸 처리해야 함 — v1에서 버그였음).
- 데이터 위생 문제 실존: `16PM`, `???`, 대소문자 혼용, 헤더에 "AM/PM 구분 해주세요" 메모.

### 이번 세션에서 확인된 운영 마찰
- 옵션이 11개+ 로 늘어남 — 매주 만지는 건 사실상 0개 (한 번 정하면 끝).
- localStorage 상태가 브라우저마다 달라서 **공유 불가** + 마이그레이션 지옥(dv 해킹까지 동원).
- 카드 레이아웃 요구는 결국 하나로 수렴: **고정 60px 직사각형, 이름 좌상단, 시간 우상단, 넘치면 글자 축소**.
- 이미지는 외부 URL만 가능 → 운영진에게 "이미지를 어디에 올리나" 안내가 필요.

## 3. 출력 방식 재검토 (원점에서)

| 방식 | 판정 | 근거 |
|---|---|---|
| **A. HTML 표 붙여넣기 (현재)** | ✅ **유지** | 스파이크로 링크·색·이미지 생존 확인. 주간 비용 = 복사+붙여넣기 1회. |
| B. 큰 이미지 1장 + 이미지맵 | ❌ 기각 | 이미지맵 미작동 사례 다수(v1 설계문서), 검증 비용 > 이득 |
| C. 칸별 이미지 + 개별 링크 | ❌ 기각 | 비주얼은 최고지만 매주 30여 개 칸 수동 업로드+링크 = 프리미스("주간 비용 ≈ 0") 위반 |
| D. 외부 페이지로 보내는 문 | ❌ 기각 | "방문자를 카페 밖으로 안 내보낸다" — v1부터 불변 제약 |

**결론: A 확정.** 재검토했지만 실측이 A를 지지한다. 유일한 보완: 텍스트 링크의 클릭 가시성(밑줄)은 유지.

## 4. 핵심 재설계 — "상태를 시트로, 도구를 URL로"

운영진 공유의 장애물은 딱 하나: **상태가 내 브라우저(localStorage)에 있다.**
색·URL·이미지·테마를 A가 바꿔도 B의 화면엔 없다. 도구 파일을 카톡으로 돌리는 것도 버전 지옥.

### 결정 1: 구글시트 = 유일한 진실
이미 운영진 전원이 공유하는 시트가 있다. 데이터만이 아니라 **설정도 시트에 넣는다**:
```
시트 탭 구성:
  0615 ~ 0621, 0622 ~ 0628, ...   ← 주차 (기존 그대로)
  members                          ← 신규: 멤버 | 배경색 | 글자색 | 채널URL | 이미지URL
  config (선택)                    ← 신규: 헤더문구 | 배지문구 | 로고URL 등 테마
```
- 도구는 **무상태**가 된다: 열면 시트에서 전부 읽어온다. 누가 열어도 같은 결과.
- 색 바꾸기 = 시트에서 셀 하나 수정. 운영진 전원에게 즉시 반영. 마이그레이션 문제 소멸.
- 도구 안 "멤버/디자인" 편집 UI는 **얇게 유지하되 "시트에 저장하세요" 안내로 전환** (임시 미리보기용).

### 결정 2: GitHub Pages로 호스팅
도구는 이미 빌드 산출물이 단일 HTML이다. 그대로 GitHub Pages에 올리면:
- 운영진은 **URL 하나** 북마크. 설치·파일전달·버전관리 전부 소멸.
- 업데이트 = push 한 번, 전원 자동 최신.
- gviz JSONP는 https 페이지에서도 동작(이미 확인된 방식).
- 파일로도 여전히 동작(오프라인 폴백 유지).

### 결정 3: 매주 루틴을 UI의 전부로
매주 실제로 하는 일은 3가지뿐: **주 고르기 → (오타 수정) → 복사**.
```
메인 화면 = [주 탭들] [미리보기(=결과)] [HTML 복사]  ← 이게 전부
설정 화면 = 멤버·테마 (시트 기반, 가끔만 엶)
```
- 디자인 토글 11개 → **프리셋 2~3개**(기본 직사각형 / 둥근 포스터)로 접고, 세부는 "고급" 아래로.
- 확정된 카드 규칙(60px 직사각형, 이름 좌상단·시간 우상단, auto-fit 글자 축소)은 **옵션이 아니라 기본 동작**.

### 결정 4: 파서는 관대하게, 경고는 크게
시트 위생은 통제 불가(여러 명이 적음). 파서가 고치려 들지 말고:
- `16pm`·`2 PM`·`???` 등은 최대한 해석하되, 해석 못 한 칸은 **복사 전에 경고 배지로 표시** ("화요일 눈요: 시간 '???' — 이 칸은 맨 아래 정렬됨").
- 링크 없는 칸 수를 복사 버튼 옆에 상시 표시 (지금의 "링크 N" 확장).

## 5. 화면 설계

```
┌────────────────────────────────────────────────────────┐
│ 주간 스케줄 대문 만들기        [설정⚙]  [HTML 복사]     │
├────────────────────────────────────────────────────────┤
│ ‹ 0615  0622  [0629~0705]  ＋ ›      ⚠ 시간해석실패 1   │
├────────────────────────────────────────────────────────┤
│                                                        │
│              미리보기 (740px, = 붙여넣을 결과)           │
│              칸 클릭 → 그 자리에서 제목/시간/링크 수정     │
│                                                        │
├────────────────────────────────────────────────────────┤
│ 복사 → 카페 관리 → 대문 → HTML 편집 → 붙여넣기 → 저장    │
└────────────────────────────────────────────────────────┘
```
- 편집 리스트(현재 방식)는 유지하되 **미리보기 직접 클릭 편집**을 1차 동선으로 승격
  (운영진은 표를 보고 고치지, 행 목록을 스캔하지 않는다).
- 설정⚙ = members/config 시트 상태 보여주기 + 임시 override. "영구 저장은 시트에서" 링크.

## 7. 성공 기준
- 운영진 중 **내가 아닌 사람**이 URL만 받고 5분 안에 이번 주 대문을 올린다.
- 매주 손대는 단계 ≤ 3 (시트 수정 → 복사 → 붙여넣기). 도구 설정은 0단계.
- 색/멤버 변경이 시트 수정만으로 전원에게 반영된다.

## 8. 리스크 · 미해결
- **div height 미검증** (P0 게이트) — 실패해도 `<td height>` 폴백 있음, 설계 불변.
- gviz는 비공식 API — 구글이 막으면 "웹에 게시 CSV"로 전환 (파서 입력만 교체, 1일 작업).
- 시트 구조를 운영진이 바꾸면 파서가 깨짐 → 경고 배지가 1차 방어, `members` 탭 도입 시 구조 문서를 시트 첫 탭에 적어두기.
- CLI(`cli.mjs`)는 fitCards(글자 축소)가 없어 앱과 결과가 다름 → P1에서 "정식 경로는 웹"으로 문서화하고 CLI는 디버그용으로 강등.
